平衡二叉树

发表于 2018-05-12 | 分类于 面试算法 | | 阅读次数

问题

思路

递归计算每一个节点的深度。简单粗暴，但存在的问题是，每一个点都会被上面的点计算深度时重新递归计算一次。
判断是否平衡，采用前序遍历。若当前节点平衡，不返回值，继续向下遍历判断。直到节点为空，不断向上返回true。

优化上面的递归，发现子树不平衡，则不再计算深度，而是直接返回-1
